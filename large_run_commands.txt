Base load and SCD2 example commands
===================================

Run these from PowerShell in the repo root:

  cd F:\SyncDataGenerator_v1.0
  $env:PYTHONUTF8='1'


Example A: normal full base load, full rule validation, SCD2 generation, SCD2 validation
---------------------------------------------------------------------------------------

Run #1 creates the first base load:

  .\venv\Scripts\python.exe .\main.py

Validate all rules and churn rules for Run #1:

  .\venv\Scripts\python.exe .\misc\transform_all_raw_to_silver.py
  .\venv\Scripts\python.exe .\misc\verify_all_silver.py

Run #2 creates the next base load and generates SCD2 outputs by comparing against Run #1:

  .\venv\Scripts\python.exe .\main.py

Validate all rules and churn rules for Run #2:

  .\venv\Scripts\python.exe .\misc\transform_all_raw_to_silver.py
  .\venv\Scripts\python.exe .\misc\verify_all_silver.py

Validate SCD2 deltas:

  .\venv\Scripts\python.exe .\misc\compare_all_scd2.py


What this checks
----------------

main.py checks base output:

  - DDL/file-column alignment
  - referential integrity
  - policy date rules
  - lead-to-policy timeline rules

verify_all_silver.py checks full silver rules:

  - relationship and uniqueness rules
  - person/contact/identity/address rules
  - lead, quote, policy, customer, and account rules
  - timeline rules
  - churn premium movement logic and distributions
  - churn claim-count logic and distributions
  - policy-cycle / tenure logic and distributions
  - policy add-on logic and distributions
  - marketing proxy logic and distributions where sat_marketing_preference exists
  - driver-experience proxy from birth date
  - vehicle-model churn segment logic and distributions

compare_all_scd2.py checks SCD2:

  - source raw SCD2 deltas
  - synthetic base SCD2 deltas
  - changed rows
  - changed columns per file
  - whether only expected SCD2 mutation columns changed


Example B: large 10M base-load only, with base validation
--------------------------------------------------------

Use this when you only want a large unique base load:

  .\venv\Scripts\python.exe .\main.py --streaming-base --total-people 10000000 --chunk-size 100000

Validate the large streaming base output:

  @'
  from config.runConfig import DDL_JSON_PATH, OUTPUT_BASE
  from validators.file_cols_validator import check_file_and_cols
  print(check_file_and_cols(DDL_JSON_PATH, OUTPUT_BASE))
  '@ | .\venv\Scripts\python.exe -

  @'
  from config.runConfig import OUTPUT_BASE
  from validators.integrity_checker import validate_integrity
  print(validate_integrity(OUTPUT_BASE))
  '@ | .\venv\Scripts\python.exe -

Note: full raw/silver/churn/SCD2 validation at 10M is not streaming-safe yet. Use Example A for the complete full-rule and SCD2 workflow.
