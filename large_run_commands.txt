Base load and SCD2 example commands
===================================

Run these from PowerShell in the repo root:

  cd F:\SyncDataGenerator_v1.0
  $env:PYTHONUTF8='1'

Detailed current rules are documented in docs/current_rules_reference.md.
Scenario config meanings are documented in docs/scenario_config_reference.md.


Example A: normal synthetic base load, rule validation, SCD2 generation, SCD2 validation
--------------------------------------------------------------------------------------

Run #1 creates the first base load:

  .\venv\Scripts\python.exe .\main.py

By default, this skips raw/canonical/silver and data/new_outputs_src generation to keep runs faster.
It creates base output, synthetic/base, synthetic/enhanced, synthetic/mlops, and synthetic SCD2 where prior runs exist.

To include raw/canonical/silver outputs:

  .\venv\Scripts\python.exe .\main.py --include-raw-silver

To include raw/canonical/silver plus new_outputs_src:

  .\venv\Scripts\python.exe .\main.py --include-raw-silver --include-new-outputs-src

Validate all rules and churn rules for Run #1:

  .\venv\Scripts\python.exe .\validate_churn_kpis.py
  .\venv\Scripts\python.exe .\misc\verify_enhanced_synthetic.py
  .\venv\Scripts\python.exe .\misc\verify_mlops_synthetic.py
  .\venv\Scripts\python.exe .\misc\verify_mlops_churn_kpis.py

Run #2 creates the next base load and generates SCD2 outputs by comparing against Run #1:

  .\venv\Scripts\python.exe .\main.py

Validate all rules and churn rules for Run #2:

  .\venv\Scripts\python.exe .\validate_churn_kpis.py
  .\venv\Scripts\python.exe .\misc\verify_enhanced_synthetic.py
  .\venv\Scripts\python.exe .\misc\verify_mlops_synthetic.py
  .\venv\Scripts\python.exe .\misc\verify_mlops_churn_kpis.py

Validate SCD2 deltas:

  .\venv\Scripts\python.exe .\misc\compare_all_scd2.py


What this checks
----------------

main.py checks base output:

  - DDL/file-column alignment
  - referential integrity
  - policy date rules
  - lead-to-policy timeline rules

verify_all_silver.py checks full silver rules only when raw/silver outputs were generated with --include-raw-silver:

  - relationship and uniqueness rules
  - person/contact/identity/address rules
  - lead, quote, policy, customer, and account rules
  - timeline rules
  - churn premium movement logic and distributions
  - churn claim-count logic and distributions
  - policy-cycle / tenure logic and distributions
  - policy-cycle churn direction: higher completed tenure has lower churn
  - policy-cycle churn ranges from the workbook: <1 year 35-50%, 1-2 years 25-35%, 3-5 years 15-25%, >5 years 8-15%
  - policy add-on logic and distributions
  - sales-channel churn variance, with AGENT carrying broker/aggregator-like higher churn; the workbook does not define a channel benchmark range
  - marketing proxy logic and distributions where sat_marketing_preference exists
  - driver-experience proxy from birth date
  - vehicle-model churn segment logic and distributions

Churn distributions are configured in config/scenario_v1.json under churn_settings.
Each config key is explained in docs/scenario_config_reference.md.

compare_all_scd2.py checks SCD2:

  - synthetic base SCD2 deltas
  - synthetic enhanced SCD2 deltas
  - synthetic MLOps SCD2 deltas
  - source raw SCD2 deltas when --include-raw-silver was used
  - changed rows
  - changed columns per file
  - whether only expected SCD2 mutation columns changed


Example B: large 10M base-load only, with base validation
--------------------------------------------------------

Use this when you only want a large unique base load:

  .\venv\Scripts\python.exe .\main.py --streaming-base --total-people 10000000 --chunk-size 100000

Validate the large streaming base output:

  @'
  from config.runConfig import DDL_JSON_PATH, OUTPUT_BASE
  from validators.file_cols_validator import check_file_and_cols
  print(check_file_and_cols(DDL_JSON_PATH, OUTPUT_BASE))
  '@ | .\venv\Scripts\python.exe -

  @'
  from config.runConfig import OUTPUT_BASE
  from validators.integrity_checker import validate_integrity
  print(validate_integrity(OUTPUT_BASE))
  '@ | .\venv\Scripts\python.exe -

Validate only the churn KPI fields in the latest base run:

  .\venv\Scripts\python.exe .\validate_churn_kpis.py

Or validate a specific base run folder:

  .\venv\Scripts\python.exe .\validate_churn_kpis.py --path .\data\synthetic\base\<run_id>

Note: full raw/silver validation at 10M is not streaming-safe yet. Use Example A for the complete synthetic and SCD2 workflow.
